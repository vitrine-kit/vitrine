---
description: Первичная настройка проекта — зависимости, env, запуск dev-сервера
---

Подними этот Vitrine-проект локально. Действуй по шагам, останавливайся на ошибках:

1. Проверь Node >= 20 (`node -v`). Если ниже — попроси обновить (Node 20 LTS).
2. `pnpm install` (пакеты `@vitrine-kit/*` публичны в npm — токен не нужен).
3. Если нет `.env` — `cp .env.example .env`. Пройдись по ключам и подскажи, что заполнить
   (для прода обязателен реальный `DATABASE_URL`; в dev есть SQLite-fallback).
4. Запусти dev-сервер; backend-специфику смотри в `README.md`:
   - Payload: `pnpm dev` → витрина `http://localhost:3000`, админка `/admin`.
   - Vendure: `pnpm vendure` (Shop API на :3001, отдельный терминал) + `pnpm dev` (:3000).
5. Убедись, что страница открывается. Кратко доложи статус и следующий шаг
   (`/design` для брендирования или `/add-feature` для фич).
